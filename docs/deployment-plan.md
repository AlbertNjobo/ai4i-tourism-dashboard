# Deployment Plan

| Field | Response |
|-------|----------|
| **Deployment environment** | Cloud (static hosting — Vercel, Netlify, or GitHub Pages) |
| **Hosting provider** | Vercel (Next.js optimized) or Netlify |
| **Operator** | Tourism authority IT team or POTRAZ digital division |
| **Pilot site** | Zimbabwe Tourism Authority, Harare — internal team of 5–10 analysts |
| **Users to onboard** | 5–10 tourism authority analysts and destination managers |
| **Training and support** | 1-hour onboarding session + written user guide + WhatsApp support channel |
| **Monitoring** | Vercel Analytics for usage, Lighthouse for performance, manual quarterly review |
| **Backup and recovery** | Git version control, CSV data in repository, Vercel automatic backups |
| **Connectivity plan** | Fully static — works offline after initial load, low bandwidth friendly |
| **Scale pathway** | Phase 1: Single authority pilot → Phase 2: Multi-authority deployment → Phase 3: Real data integration |
| **Milestones** | 30 days: Clean codebase, confirm pilot partner → 60 days: Run pilot, collect feedback → 90 days: Prepare adoption case |
